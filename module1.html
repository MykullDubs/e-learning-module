Week 1: Understanding Prompt Structure

Giving the AI good directions is the key to getting good results.

The Core Components of a Great Prompt

Think of a prompt as a recipe you're giving to a chef. The more precise and detailed the recipe, the better the final dish. A great prompt for code generation typically includes four key ingredients:

1. Role

Tell the AI who it should be. "Act as an expert frontend developer" sets the context and influences the style and quality of the response.

2. Task

Clearly state what you want it to do. "Create a single-page interactive dashboard" is a specific, actionable command.

3. Constraints & Details

Provide the specific rules and requirements. This includes technologies ("Use Tailwind CSS"), styling ("Use a warm, neutral color palette"), and content details.

4. Format

Specify the desired output format. "Provide the code in a single HTML file" ensures you get exactly what you need.

Vague vs. Structured: An Example

Let's see the difference in action. We'll ask the AI to create a simple "Contact Us" button. Click the tabs below to see how a vague prompt compares to a structured one.

The Prompt:

The Result:

This is functional, but it's unstyled and doesn't fit any design aesthetic. It's the bare minimum.

The Prompt:

ROLE: Act as a UI developer.
TASK: Create an HTML button element for a "Contact Us" call to action.
CONSTRAINTS: Use Tailwind CSS for styling. The button should have a blue background, white text, rounded corners, padding, and a subtle shadow. It should have a slightly darker blue background on hover.

The Result:

This result is exactly what we asked for: styled, interactive, and ready to be used in a real project.
