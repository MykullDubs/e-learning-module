Generating Interactive Charts

Bring data to life by telling the AI to create dynamic visualizations.

Why Use a Chart Library?

While you could theoretically ask an AI to write a chart from scratch, it's incredibly inefficient. Charting libraries like **Chart.js** do all the heavy lifting for drawing, animating, and handling interactions. Your job is to tell the AI how to use that library.

Key Prompt Components for Charts

When asking for a chart, you need to be very specific. In addition to the core components we've already learned, you must include:

1. The Library

Explicitly state which library to use. For example: "Use Chart.js to create a bar chart."

2. The Chart Type

Be specific about the visualization you want, such as 'bar', 'line', 'doughnut', or 'pie'.

3. The Data

Provide the data directly in the prompt. Give it the labels for the axes and the numerical data points to plot.

4. The HTML Container

This is critical for responsiveness. Instruct the AI to place the `` element inside a `

` container. This allows the chart to resize properly.

Here's a complete prompt that puts all these pieces together to create a simple bar chart showing weekly user engagement.

The Prompt:

TASK: Create a vertical bar chart showing user engagement for the last 7 days.
LIBRARY: Use Chart.js.
DATA:
- Labels: ['Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat', 'Sun']
- Data Points: [120, 190, 150, 210, 180, 220, 250]
- Label for the dataset: 'Active Users'
HTML STRUCTURE:
- Create a `div` with a class of 'chart-container'.
- Inside the div, create a `canvas` element with an id of 'engagementChart'.
JAVASCRIPT:
